环境准备

1. Snowflake 账号

步骤说明
注册账号访问 signup.snowflake.com 注册免费试用（30 天）或使用企业账号
创建数据库执行 scripts/init_snowflake.sql（创建 QUICKSIGHT_DB + 源表 + 示例数据）
配置 Git 集成在 init_snowflake.sql 第 6 部分，创建 Secret + API Integration + Git Repository
创建 WorkspaceSnowflake UI → Projects → Workspaces → Create from Git Repository

2. AWS MWAA 环境

步骤说明
创建 MWAA 环境AWS 控制台创建，选择 Airflow 2.10.3，配置 VPC + S3 桶
上传 requirements.txtaws s3 cp requirements/requirements.txt s3://<bucket>/requirements.txt
上传 startup.shaws s3 cp scripts/startup.sh s3://<bucket>/startup.sh
配置 Snowflake ConnectionAirflow UI → Admin → Connections → 添加 snowflake_default
等待环境就绪约 20-30 分钟，状态变为 AVAILABLE

详细指南见 docs/MWAA_SETUP_GUIDE.md

3. GitHub Actions CI/CD

步骤说明
配置 OIDC + IAM Role执行 bash scripts/init_github_actions.sh（一键配置）
验证git push 后查看 GitHub Actions 页面是否自动触发部署

4. 本地开发环境

步骤说明
安装 dbtpip install dbt-snowflake
安装 IDE 插件Kiro / VSCode 安装 dbt Power User 插件（语法高亮、编译预览、血缘图）
配置凭证cp dbt_project/.env.example dbt_project/.env 并填入 Snowflake 凭证
验证连接source dbt_project/.env && cd dbt_project && dbt debug --profiles-dir .

环境隔离

环境Target写入 Schema触发方式
🛠️ 本地开发devQUICKSIGHT_DB.DEV_*手动 dbt run
🚀 MWAA 生产prodQUICKSIGHT_DB.PUBLIC_*每日 08:00 自动

快速开始
