🧠 Skill — SQL → dbt 智能翻译

Snowflake Notebook SQL 自动转换为标准 dbt 模型，包含 ref/source/config/去重逻辑。

# .kiro/skills/sql-to-dbt.md Snowflake SQL → Kiro Skill → dbt 模型 ✓ 自动 ref()/source() 引用 ✓ 字段映射校验（staging 输出 vs marts 引用） ✓ 分层规范（materialized/schema/tags） ✓ 移除 ORDER BY/LIMIT

🛡️ 数据质量：Snowflake DMF

ODS 层自动数据质量监控，Snowpipe 加载后触发检测。

表类型dbt 策略DMF 监控
CUSTOMERS维度表merge (upsert)NULL_COUNT(customer_id, email) + DUPLICATE_COUNT(customer_id)
ORDERS事实表append (追加)NULL_COUNT(order_id, customer_id, total_amount, status)
ORDER_ITEMS事实表append (追加)NULL_COUNT(order_item_id, order_id, quantity, unit_price)
维度表检查唯一性（DUPLICATE_COUNT），事实表只检查完整性（NULL_COUNT）
触发方式：TRIGGER_ON_CHANGES | 配置脚本：scripts/init_dmf.sql

🔄 dbt 转换能力

能力实现说明
增量物化incremental + merge/append维度表 upsert，事实表按天追加
模块化ref() / source()模型间依赖自动管理
Slim CIstate:modified+PR 时只跑修改的模型，节省 ~30% 成本
成本归因query_tag按层追踪 Snowflake credit 消耗
数据测试unique / not_null每次运行自动验证
自动授权on-run-end HookGRANT ANALYST_ROLE 读 Gold 层
Owner 追踪set_owner_tag模型责任人标签

环境准备

1. Snowflake 账号

步骤说明
注册账号访问 signup.snowflake.com 注册免费试用或使用企业账号
初始化环境执行 scripts/init_snowflake.sql（数据库 + 表 + 角色 + Git 集成）
创建 WorkspaceSnowflake UI → Projects → Workspaces → Create from Git Repository

2. AWS MWAA

步骤说明
创建环境AWS 控制台，Airflow 2.10.3，配置 VPC + S3
上传配置requirements.txt + startup.sh → S3
配置连接Airflow UI → Connections → snowflake_default（DBT_USER + DBT_ROLE）

3. CI/CD

步骤说明
GitHub Actionsbash scripts/init_github_actions.sh（OIDC + IAM Role 一键配置）

4. 本地开发

步骤说明
安装 dbtpip install dbt-snowflake
IDE 插件安装 dbt Power User（语法高亮、编译、血缘图）
配置凭证cp .env.example .env → 填入 Snowflake 凭证
验证dbt debug --profiles-dir . → All checks passed!
